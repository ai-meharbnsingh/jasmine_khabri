       - Get query from `update.callback_query`
       - MUST call `await query.answer()` first (dismisses Telegram loading indicator)

         - "menu_status": Call fetch_pipeline_status(), format same as status_command text, call query.edit_message_text(text)
         - "menu_help": Build help text (same content as help_command), call query.edit_message_text(text)
       - Wrap each branch in try/except: on failure, edit message to "Failed to load [section]. Try again."

    IMPORTANT anti-patterns to avoid:
    - Do NOT use `update.message.reply_text()` in callback handler -- callback queries do NOT have `update.message`. Use `query.edit_message_text()`.
    - Do NOT forget `query.answer()` -- Telegram clients show loading spinner indefinitely without it.
