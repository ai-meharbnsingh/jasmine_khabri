<task type="auto" tdd="true">
  <name>Task 1: Priority-based article selector</name>
  <files>src/pipeline/deliverers/selector.py, tests/test_selector.py</files>
  <behavior>
    - select_articles([15 NEW articles: 10 HIGH, 3 MED, 2 LOW]) -> (high=8, medium=3, low=2) — HIGH capped at 8
    - select_articles([5 NEW articles: 2 HIGH, 2 MED, 1 LOW]) -> (high=2, medium=2, low=1) — all included when under 15
    - select_articles([20 NEW articles: 3 HIGH, 10 MED, 7 LOW]) -> (high=3, medium=10, low=2) — fill from MEDIUM surplus
    - select_articles([articles with mixed dedup_status]) -> only NEW articles considered, DUPLICATE/UPDATE ignored
    - select_articles([]) -> ([], [], []) — empty input returns empty tuples
    - select_articles([5 HIGH only, 0 MED, 0 LOW]) -> (high=5, medium=[], low=[]) — missing tiers return empty lists
    - select_articles([0 HIGH, 20 MED, 0 LOW]) -> (high=[], medium=15, low=[]) — single tier fills to cap
    - Total across all three lists never exceeds max_stories (default 15)
  </behavior>
  <action>
    Create `src/pipeline/deliverers/selector.py` with a `select_articles` function following the project's functional style (no mutation, returns results).

    Algorithm:
    1. Filter to only dedup_status="NEW" articles
    2. Separate into HIGH/MEDIUM/LOW buckets by priority field
    3. Take up to 8 HIGH-priority articles
    4. Take all available MEDIUM (at least 4 if available)
    5. Take all available LOW (at least 2 if available)
    6. If total exceeds max_stories, trim from lowest priority first (LOW then MEDIUM)
    7. If total < max_stories and articles remain in any bucket, backfill from highest-priority surplus
    8. Return tuple of (high, medium, low) lists

    Function signature:
    ```python
    def select_articles(
        articles: list[Article],
        max_stories: int = 15,
    ) -> tuple[list[Article], list[Article], list[Article]]:
    ```

    Import Article from pipeline.schemas.article_schema. Log selection counts at INFO level.
  </action>
  <verify>
    <automated>uv run pytest tests/test_selector.py -x -v</automated>
  </verify>
  <done>select_articles correctly allocates articles by priority with all edge cases handled. All tests pass.</done>
</task>

  <name>Task 2: Telegram HTML message formatter with chunking</name>
  <files>src/pipeline/deliverers/telegram_sender.py, tests/test_telegram_sender.py</files>
  <behavior>
    - _escape_html("AT&amp;T <Corp>") -> "AT&amp;amp;T &amp;lt;Corp&amp;gt;" — escapes &, <, > in that order
    - _escape_html("normal text") -> "normal text" — no change for safe text
    - format_article_html(article_with_all_fields, 1) -> HTML with bold title, italic source+location, summary, entity line (budget+authority), Read link
    - format_article_html(article_no_entities, 3) -> HTML WITHOUT entity line — omitted when all entity fields empty
    - format_article_html(article_no_summary, 5) -> HTML WITHOUT summary line — omitted when summary empty
    - format_article_html(article_with_ampersand_url, 1) -> href has &amp; escaped in URL
    - get_delivery_period() returns "Morning" when IST hour < 12, "Evening" otherwise
    - format_delivery_message(high=[2], med=[1], low=[1], period="Morning") -> full HTML message with header (story counts), section headers with circle emojis, continuous numbering (1-2 HIGH, 3 MED, 4 LOW), separators, footer with next delivery time
    - format_delivery_message with empty tier -> that section omitted entirely (no empty "HIGH PRIORITY (0 stories)" header)
    - chunk_message(header, article_blocks, footer, max_chars=4096) -> single chunk when under limit
    - chunk_message(header, many_article_blocks, footer, max_chars=200) -> multiple chunks, never splitting mid-article, header in first chunk only, footer in last chunk only
  </behavior>
  <action>
    Implement formatting functions in `src/pipeline/deliverers/telegram_sender.py` (replacing the empty placeholder).

    Functions to implement:

    1. `_escape_html(text: str) -> str` — escape &, <, > for Telegram HTML. Order matters: & first, then < and >.

    2. `format_article_html(article: Article, number: int) -> str` — format single article per CONTEXT.md approved mockup:
       - Line 1: "{number}. <b>{escaped_title}</b>"
       - Line 2: "   <i>{escaped_source}</i>" — append " * {location}" if location non-empty
       - Line 3 (conditional): "   {escaped_summary}" — only if summary non-empty
       - Line 4 (conditional): "   Budget: {budget} * Authority: {authority}" — only if at least one entity field non-empty. Show only present fields.
       - Line 5: '   <a href="{escaped_url}">Read</a>'
       Use simple "1." numbering (not keycap emojis) per research recommendation.

    3. `get_delivery_period() -> str` — return "Morning" if current IST hour < 12, else "Evening". Use `timezone(timedelta(hours=5, minutes=30))` for IST.

    4. `format_delivery_message(high: list[Article], medium: list[Article], low: list[Article], period: str = "") -> list[str]` — build the full message and return as list of article block strings (one per article). Also build header and footer. Use:
       - Header template per CONTEXT.md: newspaper emoji, bold "Khabri {period} Brief", IST time + date, story count breakdown, horizontal separator
       - Section headers: red circle for HIGH, yellow circle for MEDIUM, green circle for LOW — with bold label and count. Skip sections with 0 articles.
       - Continuous numbering across all sections
       - Footer: separator + "Powered by Khabri" + "Next: {next_time}" (Morning->4:00 PM, Evening->7:00 AM)
       - Call `chunk_message` to split if needed.
       - Return the final list of message strings ready to send.

    5. `chunk_message(header: str, article_blocks: list[str], footer: str, max_chars: int = 4096) -> list[str]` — split at article boundaries. Header in first chunk, footer in last chunk. Each chunk must be under max_chars.

    Import Article from pipeline.schemas.article_schema. Use datetime.UTC for time operations (project convention from Phase 2).
  </action>
