  <files>

  <behavior>
    - compute_stats with 10 entries across 3 days returns correct total (10), by_date (3 entries), top_sources (top 5)
    - compute_stats with entries older than 7 days excludes them from count
    - compute_stats with entries having duplicate title_hash counts them in duplicates_prevented
    - compute_stats with empty SeenStore returns total_articles=0, empty by_date, empty top_sources
    - compute_stats with days=3 only includes last 3 days
    - format_stats_message formats dates sorted, sources ranked by count, shows total and dupes
    - format_stats_message with zero articles returns a "no data" line
  </behavior>
  <action>
    Create src/pipeline/bot/stats.py with pure functions:

    compute_stats(history: SeenStore, days: int = 7) -> dict:
    1. Calculate cutoff = datetime.now(UTC) - timedelta(days=days)
    2. Filter entries where seen_at >= cutoff ISO string (compare as strings since both are ISO 8601)
    3. Count by date: Counter(entry.seen_at[:10] for entry in recent_entries)
    4. Count by source: Counter(entry.source for entry in recent_entries)
    5. Count duplicates: total entries minus unique title_hash count (entries with same title_hash = duplicates prevented)
    6. Return dict: {"total_articles": int, "duplicates_prevented": int, "by_date": dict (date->count), "top_sources": list of (source, count) tuples top 5, "days_covered": int}

    format_stats_message(stats: dict) -> str:
    1. Header: "Delivery Statistics (Last {days} Days)\n"
    2. Summary: "Total articles processed: {N}\nDuplicates prevented: {N}\n"
    3. "Articles by date:" section with sorted dates and counts
    4. "Top sources:" section with source name and count
    5. If total_articles == 0: return "No delivery data for the last {days} days."

    Write tests in tests/test_bot_stats.py:
    - TestComputeStats: normal case, old entries excluded, duplicates counted, empty history, custom days
    - TestFormatStatsMessage: formatted output with data, empty data message
    Use class-based test pattern. Create SeenEntry fixtures with known dates/sources.
  </action>

  </verify>
  <done>

    Empty history returns sensible "no data" message.
