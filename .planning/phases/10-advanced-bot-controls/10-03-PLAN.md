    Create src/pipeline/bot/nlp.py:

    NLIntent Pydantic model (from research):
    - intent: Literal["pause", "resume", "schedule_modify", "event_schedule", "keyword_add", "keyword_remove", "unknown"]
    - confidence: float = 0.0
    - slot: str = "" (morning/evening/all)
    - duration: str = "" ("3 days", "a week")
    - new_time: str = "" ("06:30" for schedule_modify)
    - event_name: str = ""
    - event_date: str = "" (ISO date)
    - interval_minutes: int = 0
    - start_time: str = "" (HH:MM IST)
    - end_time: str = "" (HH:MM IST)
    - category: str = "" (for keyword intents)
    - keyword: str = "" (for keyword intents)

    NL_SYSTEM_PROMPT (from research, refined): System prompt explaining the Khabri bot's intents and expected JSON output format. Include examples of each intent for Claude to learn from.

    parse_nl_intent(text: str) -> NLIntent:
    1. Create anthropic.Anthropic client with ANTHROPIC_API_KEY from env
    2. Call client.messages.create with model="claude-haiku-4-5-20250315", max_tokens=256, system=NL_SYSTEM_PROMPT, user message=text
    3. Parse response text as JSON into NLIntent
    4. On any error (API failure, JSON parse error): return NLIntent(intent="unknown")

    nl_command_handler(update, context) async handler:
    1. Get message text. If len < 6 chars: ignore silently (return without reply — prevents "ok", "hi" etc.)
    2. Send "Processing..." reply (immediate feedback before AI call)
    3. Call parse_nl_intent(text) — NOTE: this is a sync Anthropic call, use asyncio.get_event_loop().run_in_executor(None, parse_nl_intent, text) to avoid blocking the event loop
    4. If confidence < 0.6: reply "I didn't understand that. Try /help for available commands."
    5. Dispatch by intent:
       - "pause": Build synthetic /pause message text with duration, call pause logic. Reply with "Understood: pausing {slot} deliveries for {duration}." Then execute the pause via GitHub write (reuse the read-mutate-write pattern from pause.py but with slot-specific paused_slots).
       - "resume": Execute resume logic. Reply "Understood: resuming deliveries."
       - "schedule_modify": Call schedule modification handler (from schedule.py). Reply with confirmation.
       - "event_schedule": Call event creation handler (from schedule.py). Reply with confirmation.
       - "keyword_add": Use keyword add logic from keywords.py pattern. Reply with confirmation.
       - "keyword_remove": Use keyword remove logic. Reply with confirmation.
       - "unknown": Reply "I didn't understand that. Try /help for available commands."

    IMPORTANT: For the NL dispatch of pause intents with specific slots (e.g., "stop evening alerts"), update the PauseState with the specific slot (["evening"]) instead of ["all"]. This is what differentiates NL from the basic /pause command.

    Write tests in tests/test_bot_nlp.py:
    - TestNLIntent: model creation and validation for each intent type
    - TestParseNLIntent: mock anthropic client, test each intent classification, test API failure fallback
    - TestNLCommandHandler: short message ignored, Processing feedback sent, dispatch to correct handler, unknown intent reply
    Mock anthropic.Anthropic using unittest.mock.patch. Mock the messages.create response.

  <action>
    Create src/pipeline/bot/schedule.py:

    parse_ist_time(text: str) -> tuple[int, int] | None (from research code example):
    - Regex pattern: r"(\d{1,2})(?::(\d{2}))?\s*(AM|PM|am|pm)?"
    - Handle AM/PM conversion (12 PM = 12, 12 AM = 0, 3 PM = 15)
    - Validate 0-23 hour, 0-59 minute
    - Return (hour, minute) in 24h format or None

    ist_to_utc_cron(hour_ist: int, minute_ist: int) -> tuple[int, int] (from research):
    - IST = UTC + 5:30
    - Subtract 330 minutes from total IST minutes
    - Wrap negative values by adding 1440
    - Return (hour_utc, minute_utc)

    schedule_command(update, context) async handler:
    1. Parse text after /schedule. If empty: show current schedule (read bot_state.json, show morning_ist and evening_ist or "default" if empty, plus active events list).
    2. Detect slot keyword: if text starts with "evening" or "ev", slot="evening", strip prefix. Else slot="morning" (default).
    3. Parse remaining text with parse_ist_time. If None: reply "Could not parse time. Try: /schedule 6:30 AM or /schedule evening 5 PM"
    4. Read bot_state.json with SHA, parse BotState
    5. Update custom_schedule: if morning, set morning_ist=f"{hour:02d}:{minute:02d}"; if evening, set evening_ist=...
    6. Write back via GitHub API
    7. Compute UTC equivalent via ist_to_utc_cron for informational reply
    8. Reply: "Updated {slot} delivery to {HH:MM} IST ({HH:MM} UTC). Note: GitHub Actions cron may need manual update to {minute} {hour} * * *"

    create_event_schedule(name: str, date: str, interval_minutes: int, start_time_ist: str, end_time_ist: str, token: str, owner: str, repo: str) -> bool:
    1. Validate params: name non-empty, date parseable, interval > 0, times valid
    2. Read bot_state.json with SHA, parse BotState
    3. Create EventSchedule(name=name, date=date, interval_minutes=interval_minutes, start_time_ist=start_time_ist, end_time_ist=end_time_ist)
    4. Append to events list: model_copy(update={"events": bot_state.events + [event]})
    5. Write back via GitHub API
    6. Return success bool

    Write tests in tests/test_bot_schedule.py:
    - TestParseIstTime: AM/PM conversion, 24h format, noon/midnight edge cases, invalid input
    - TestISTToUTC: standard conversions, day-wrap case (0:30 IST)
    - TestScheduleCommand: morning update, evening update, show current, parse failure, GitHub failure
    - TestCreateEventSchedule: successful creation, invalid params, GitHub write failure
    Mock GitHub functions using unittest.mock.patch.
