**GREEN phase: Implement `src/pipeline/fetchers/rss_fetcher.py`:**

1. `_struct_time_to_iso(st) -> str | None` — convert feedparser's `time.struct_time` (UTC) to ISO 8601 UTC string. Use `calendar.timegm(st)` (NOT `time.mktime` which uses local timezone) then `datetime.fromtimestamp(ts, tz=UTC).isoformat()`. Return None if st is None.

2. `fetch_rss_feed(url: str, source_name: str, timeout: float = 10.0) -> tuple[list[Article], str | None]` — per RESEARCH.md Pattern 1 (fetch-then-parse):
   - Create `httpx.Client(timeout=timeout)`, call `.get(url, follow_redirects=True)`, then `.raise_for_status()`
   - Parse `response.content` with `feedparser.parse()`
   - If `feed.bozo`, log warning with source_name, bozo_exception, entry count — but continue
   - For each entry: skip if no `entry.get("link", "")`, use `_struct_time_to_iso(entry.get("published_parsed"))` or fall back to now_iso, create Article with summary=""
   - Catch httpx.TimeoutException, httpx.HTTPStatusError, httpx.RequestError, and bare Exception (# noqa: BLE001)
   - Return `(articles, None)` on success, `([], error_string)` on failure

3. `fetch_all_rss(feeds: list[RssFeedConfig]) -> tuple[list[Article], list[dict]]` — iterate enabled feeds, call fetch_rss_feed for each, collect articles and health dicts `{"source": name, "status": "OK"/"FAIL", "count": N, "error": err_or_None}`. Log health summary table at end using stdlib logging (Pattern 6 from RESEARCH.md). Return (all_articles, health_results).

4. Update `src/pipeline/fetchers/__init__.py` to export `fetch_rss_feed` and `fetch_all_rss`.

**IMPORTANT implementation notes:**
- Use `calendar.timegm()` NOT `time.mktime()` for struct_time to UTC timestamp. `mktime` uses local timezone which causes wrong UTC conversion.
- Use `entry.get("title", "").strip()` and `entry.get("link", "")` — feedparser entries support dict-style access.
- Follow ruff rules: imports sorted (I), f-strings preferred over % format for string building (but logging.info uses % for lazy evaluation — that's correct).
- `# noqa: BLE001` on the broad `except Exception` — matches existing project pattern in main.py.

Run tests again — they must PASS.
