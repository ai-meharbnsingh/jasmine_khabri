  <name>Task 2: /pause and /resume command handlers</name>
  <files>
    src/pipeline/bot/pause.py,

  </files>
  <behavior>
    - pause_command with "3 days" sets paused_until to now+3days UTC ISO, paused_slots=["all"], writes to GitHub
    - pause_command with no args sets paused_until="" (indefinite), paused_slots=["all"]
    - pause_command replies with confirmation including expiry time in IST
    - resume_command clears paused_until and paused_slots, writes to GitHub
    - resume_command when not paused replies "Deliveries are not paused"
    - pause_command with GitHub write failure replies with error message
    - resume_command with GitHub write failure replies with error message
    - Both handlers read GITHUB_PAT, GITHUB_OWNER, GITHUB_REPO from env (same pattern as keywords.py)
  </behavior>
  <action>
    Add to src/pipeline/bot/pause.py:

    pause_command(update, context) async handler:
    1. Extract text after /pause from update.message.text (strip "/pause" prefix)
    2. If text provided, call parse_duration(text). If None, reply "Could not parse duration. Try: /pause 3 days"
    3. Read GITHUB_PAT, GITHUB_OWNER, GITHUB_REPO from env vars (same guard as keywords.py)
    4. Read bot_state.json with SHA via read_github_file_with_sha
    5. Parse into BotState, compute paused_until:
       - With duration: datetime.now(UTC) + duration, format as ISO 8601 string
       - Without duration (no args): set paused_until="" to indicate indefinite pause
    6. Set paused_slots=["all"] (Phase 10 only supports all-or-nothing pause; per-slot is via NL in Plan 03)
    7. Write back via write_github_file with commit message "bot: pause deliveries"
    8. Reply with confirmation: "Deliveries paused until {expiry_ist}" or "Deliveries paused indefinitely. Send /resume to restart."
    9. On any error: log warning, reply with error message

    resume_command(update, context) async handler:
    1. Read env vars + bot_state.json with SHA (same as pause)
    2. If pause.paused_until == "" and pause.paused_slots == []: reply "Deliveries are not paused."
    3. Clear pause: model_copy(update={"pause": PauseState()})
    4. Write back to GitHub
    5. Reply "Deliveries resumed."
    6. On error: log + reply with error

    Use _IST = timezone(timedelta(hours=5, minutes=30)) for IST display (same constant as telegram_sender.py).

    Add handler tests to tests/test_bot_pause.py:
    - TestPauseCommand: with duration, without duration, unparseable duration, GitHub write failure, missing env vars
    - TestResumeCommand: when paused, when not paused, GitHub write failure
    Mock github.read_github_file_with_sha and github.write_github_file using unittest.mock.patch (same pattern as test_bot_keywords.py).
