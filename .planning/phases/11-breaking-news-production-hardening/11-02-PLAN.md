    **Step 1: Extend PipelineStatus schema** (`src/pipeline/schemas/pipeline_status_schema.py`):

    usage_month: str = ""                  # "YYYY-MM" for monthly reset
    monthly_deliver_runs: int = 0          # Full pipeline runs this month
    monthly_breaking_runs: int = 0         # Breaking news checks this month

    **Step 2: Add monthly reset to load_pipeline_status** (`src/pipeline/utils/loader.py`):
    Follow the same pattern as `load_ai_cost` monthly reset:
    - Get `current_month = datetime.now(UTC).strftime("%Y-%m")`
    - After loading PipelineStatus from JSON, check if `status.usage_month != current_month`
    - If different month: use `model_copy(update={...})` to reset monthly counters to 0 and set usage_month to current_month. Preserve non-monthly fields (last_run_utc, articles_fetched, etc.)
    - If same month: return as-is

    **Step 3: Update main.py** to load-then-increment run counter:
    - At the start of `run()`, load existing pipeline status: `prev_status = load_pipeline_status("data/pipeline_status.json")`
    - When constructing final PipelineStatus at the end, include:
      - `usage_month=datetime.now(UTC).strftime("%Y-%m")`
      - `monthly_deliver_runs=prev_status.monthly_deliver_runs + 1`
      - `monthly_breaking_runs=prev_status.monthly_breaking_runs` (preserve, don't reset)
      - `monthly_breaking_alerts=prev_status.monthly_breaking_alerts` (preserve)
      - `est_actions_minutes=prev_status.est_actions_minutes + 3.0` (estimated 3 min per deliver run including checkout+install)

    **Step 4: Update breaking.py** to load-then-increment breaking counter:
    - At the end of `run_breaking()`, regardless of whether alerts were sent:
      - Load: `prev_status = load_pipeline_status("data/pipeline_status.json")`
      - Compute: `alerts_sent = len(high_articles)` (or 0 if no alerts)
      - Save: `save_pipeline_status(prev_status.model_copy(update={...}), ...)`
        - `monthly_breaking_runs=prev_status.monthly_breaking_runs + 1`
        - `monthly_breaking_alerts=prev_status.monthly_breaking_alerts + alerts_sent`
        - `est_actions_minutes=prev_status.est_actions_minutes + 1.5` (estimated 1.5 min per breaking run)
        - `usage_month=datetime.now(UTC).strftime("%Y-%m")`
    - Make sure to add the `load_pipeline_status` and `save_pipeline_status` imports to breaking.py

    - In `tests/test_main.py`: add `TestRunCounter` verifying main.py increments deliver run counter
    - Breaking run counter tests can go in `tests/test_breaking.py` (add `TestBreakingRunCounter` class)
