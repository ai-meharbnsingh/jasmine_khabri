<task type="auto" tdd="true">
  <name>Task 1: Breaking news entrypoint with two-stage filter, guards, and alert formatter</name>
  <files>src/pipeline/breaking.py, tests/test_breaking.py</files>
  <behavior>
    - TestBreakingFilter: Articles with keyword score >= 80 pass fast-path filter; articles below 80 are excluded; articles with exclusion keywords are excluded
    - TestBreakingDedup: Articles already in seen.json are filtered out by dedup; only dedup_status="NEW" articles proceed; seen.json is updated with alerted articles
    - TestBreakingAIGate: When AI budget < $3.00, classify_articles is called for keyword-HIGH candidates; when budget >= $3.00, AI is skipped and keyword score trusted (priority set to HIGH directly); when AI confirms non-HIGH, article is excluded from alert
    - TestBreakingPause: When bot_state.pause.paused_slots is non-empty and pause is active (unexpired or indefinite), run_breaking returns immediately with no alerts; when pause is expired, breaking proceeds normally
    - TestBreakingTimeWindow: Breaking check skips when current IST time is within 30 minutes of 7:00 AM or 4:00 PM (scheduled delivery windows); runs normally outside those windows
    - TestBreakingFormat: format_breaking_alert produces Telegram HTML with siren emoji header "BREAKING NEWS ALERT", numbered articles with bold titles, italic sources, optional summaries, read links, and footer "Full brief in next scheduled delivery"
    - TestRunBreaking: Full run_breaking integration (mocked I/O): loads state, fetches RSS, filters, deduplicates, optionally AI-confirms, formats alert, sends via send_telegram_message to all configured chat IDs, saves seen.json
  </behavior>
  <action>
    Create `src/pipeline/breaking.py` as a lightweight entrypoint (NOT a copy of main.py) that reuses existing modules.

    The module must contain:

    1. `_BREAKING_HIGH_THRESHOLD = 80` — same as keyword fallback HIGH from Phase 5
    2. `_BREAKING_AI_BUDGET_RESERVE = 3.00` — reserve $2 for scheduled runs

    3. `format_breaking_alert(articles: list[Article]) -> str` — formats breaking news for Telegram:
       - Header: siren emoji + bold "BREAKING NEWS ALERT" + article count + separator line
       - Each article: numbered, bold escaped title, italic escaped source, optional escaped summary, "Read" link
       - Footer: separator + "Full brief in next scheduled delivery"
       - Uses `_escape_html` from telegram_sender.py

    4. `breaking_filter(articles: list[Article], keywords: KeywordsConfig, ai_cost: AICost) -> tuple[list[Article], AICost]`:
       - Stage 1: keyword fast-path using `score_article()` — only articles with score >= 80 pass
       - Stage 2: AI confirmation using `classify_articles()` — only if `ai_cost.total_cost_usd < _BREAKING_AI_BUDGET_RESERVE`
       - When AI budget exceeded, trust keyword score: set priority="HIGH" via model_copy
       - Return only articles with priority == "HIGH" after AI confirmation
       - Return updated ai_cost

    5. `_is_delivery_window(now_ist: datetime) -> bool`:
       - Returns True if now_ist is within 30 minutes before or after 7:00 AM or 4:00 PM IST
       - Used as a guard to prevent breaking alerts colliding with scheduled delivery

    6. `_is_paused(bot_state: BotState) -> bool`:
       - Returns True if pause is active: paused_slots non-empty AND (paused_until empty for indefinite, or paused_until is in the future)
       - Checks "all" slot or any slot presence

    7. `run_breaking() -> None` — main entrypoint:
       - Load config, check breaking_news_enabled guard
       - Load bot_state, check pause guard via `_is_paused()`
       - Check delivery window guard via `_is_delivery_window()`
       - Fetch RSS only (NO GNews — save quota for scheduled runs)
       - Run keyword-only fast-path (score_article with threshold >= 80)
       - If no candidates, return early
       - Dedup against seen.json via filter_duplicates(), keep only NEW
       - If no new articles, save seen and return
       - Run breaking_filter (AI gate)
       - If no HIGH articles after AI, save seen and return
       - Format via format_breaking_alert()
       - Send via send_telegram_message() to each chat_id (env var TELEGRAM_CHAT_IDS, fallback config)
       - Save seen.json and ai_cost.json
       - Log all steps at INFO level

    8. `if __name__ == "__main__": run_breaking()` for CLI invocation

    Create `tests/test_breaking.py` with test classes matching the behavior spec above. Use `unittest.mock.patch` for all external dependencies (file I/O, Telegram API, AI classifier). Use `freezegun` or manual datetime mocking for time-window tests (check if freezegun is in deps; if not, use `unittest.mock.patch` on `datetime.now`).

    Key anti-patterns to AVOID:
    - Do NOT call GNews in breaking path (quota reserved for scheduled runs)
    - Do NOT send email for breaking news (Telegram only for speed)
    - Do NOT run geo_filter (breaking uses keyword threshold only, simpler fast-path)
    - Do NOT duplicate formatting logic from telegram_sender.py — import _escape_html
