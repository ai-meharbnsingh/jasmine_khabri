    - Test: format_overflow_notice_email returns HTML notice with overflow count
    - Test: format_slow_news_log returns log string with available count
  </behavior>
  <action>
    Create `src/pipeline/deliverers/edge_cases.py`:

    1. **`EdgeCaseResult` dataclass** (or Pydantic BaseModel for consistency):
       - `is_no_news: bool` -- zero articles
       - `is_slow_news: bool` -- fewer than max_stories articles (but not zero)
       - `has_overflow: bool` -- more HIGH articles than _HIGH_CAP (8)
       - `overflow_count: int` -- how many HIGH articles beyond cap
       - `total_available: int` -- total articles passed in

    2. **`check_edge_cases(articles: list[Article], high_cap: int = 8, max_stories: int = 15) -> EdgeCaseResult`:**
       Count NEW articles with valid priority. Count HIGH among those. Compute:
       - `is_no_news = total == 0`
       - `is_slow_news = 0 < total < max_stories`
       - `has_overflow = high_count > high_cap`
       - `overflow_count = max(0, high_count - high_cap)`

    3. **`format_no_news_telegram() -> str`:** Telegram HTML-formatted message:
       "{newspaper emoji} <b>Khabri {period} Brief</b>\n{date} | {time} IST\n{line}\n\nNo relevant infrastructure or real estate news found this cycle.\nWe'll check again at {next_time} IST.\n\n{line}\nPowered by Khabri"
       Import get_delivery_period and _IST from telegram_sender. Compute next delivery time (if morning, next is evening 4 PM; if evening, next is morning 7 AM).

    4. **`format_no_news_email() -> str`:** Minimal HTML email body with same message content, styled table, muted text.

    5. **`format_overflow_notice_telegram(overflow_count: int) -> str`:** Returns `"\n{warning emoji} {N} more HIGH-priority stories available -- reply 'more' to see them"` (HTML-escaped for Telegram).

    6. **`format_overflow_notice_email(overflow_count: int) -> str`:** Returns HTML `<tr>` with warning-style card noting overflow count. Note: "More stories available via Telegram" (email reply not supported).

    7. **`format_slow_news_log(total: int, max_stories: int) -> str`:** Returns log message: "Slow news day: {total} articles available (below {max_stories} cap)"

    Create `tests/test_edge_cases.py` with class-based pytest pattern. Test all behaviors above. Use Article fixtures with priority and dedup_status set.
