    - read_github_file_with_sha("data/keywords.yaml", token, owner, repo) returns (content_string, sha_string) on 200
    - read_github_file_with_sha raises httpx.HTTPStatusError on 404
    - write_github_file("data/keywords.yaml", new_content, "bot: add keyword", sha, token, owner, repo) returns True on 200/201
    - write_github_file returns False on API error (catches exception, logs warning)
    - add_keyword(config, "infrastructure", "bullet train") returns new KeywordsConfig with "bullet train" appended
    - add_keyword raises ValueError for unknown category
    - add_keyword raises ValueError for duplicate keyword (case-insensitive)
    - remove_keyword(config, "celebrity", "Salman Khan") returns new KeywordsConfig without that keyword
    - remove_keyword raises ValueError for unknown category
    - remove_keyword raises ValueError for keyword not found
    - serialize_keywords produces valid YAML that round-trips through KeywordsConfig
  </behavior>
  <action>
    Create `src/pipeline/bot/github.py` with two async functions:

    1. `read_github_file_with_sha(path, token, owner, repo) -> tuple[str, str]`:
       - GET `https://api.github.com/repos/{owner}/{repo}/contents/{path}`
       - Use `Accept: application/vnd.github.v3+json` (NOT raw -- need JSON to get SHA)
       - Parse response JSON: decode base64 `content` field, extract `sha` field
       - Return (decoded_content, sha)
       - Raise on non-2xx (let httpx.HTTPStatusError propagate)

    2. `write_github_file(path, content, message, sha, token, owner, repo) -> bool`:
       - PUT to same URL with base64-encoded content, sha, commit message
       - Return True on 200/201, False on any exception
       - Log warning on failure (same pattern as dispatcher.py)
       - Use httpx.AsyncClient(timeout=15.0)

    Add keyword mutation functions to a new `src/pipeline/bot/keywords.py`:

    3. `add_keyword(config: KeywordsConfig, category: str, keyword: str) -> KeywordsConfig`:
       - Case-insensitive category lookup and duplicate check
       - model_copy immutable pattern (same as GNewsQuota/AICost)
       - Raise ValueError on unknown category or duplicate

    4. `remove_keyword(config: KeywordsConfig, category: str, keyword: str) -> KeywordsConfig`:
       - Case-insensitive match for removal
       - Raise ValueError on unknown category or not-found keyword

    5. `serialize_keywords(config: KeywordsConfig) -> str`:
       - yaml.dump with default_flow_style=False, allow_unicode=True, sort_keys=False

    Write tests FIRST (RED), then implement (GREEN). Use respx for HTTP mocking (matches project pattern). Use asyncio.run() for async tests.
  </action>
