**Create `src/pipeline/fetchers/gnews_fetcher.py`:**

1. **Constants:**
   ```python
   GNEWS_BASE = "https://gnews.io/api/v4/search"
   GNEWS_MAX_PER_REQUEST = 10  # Free tier hard limit
   ```

2. **`build_gnews_queries(keywords_config: KeywordsConfig) -> list[str]`** — Per RESEARCH.md Pattern 7, construct 3-4 broad Boolean OR queries from active keyword categories. Strategy:
   - For each active category, pick the most distinctive keywords (not all 20+)
   - Group into queries of ~8-10 keywords joined by " OR "
   - Query 1: Infrastructure priority — `metro OR highway OR expressway OR NHAI OR airport OR AAI OR "smart city" OR DMRC`
   - Query 2: Regulatory — `RERA OR PMAY OR "affordable housing" OR MahaRERA OR MoHUA OR CIDCO`
   - Query 3: Real estate market — `"real estate" OR "property market" OR "housing project" OR MMRDA OR DDA`
   - Only include queries for categories that are active.
   - Quoted phrases for multi-word keywords: "smart city", "affordable housing", "real estate", etc.
   - Return list of query strings.

3. **`load_or_reset_quota(path: str | Path) -> GNewsQuota`** — Per RESEARCH.md Pattern 4:
   - Get today's UTC date as ISO string
   - Try to read and parse file as GNewsQuota
   - If file exists and date matches today, return stored quota
   - Otherwise (missing file, different date, parse error), return fresh GNewsQuota(date=today, calls_used=0, daily_limit=25)

4. **`save_quota(quota: GNewsQuota, path: str | Path) -> None`** — write model_dump_json(indent=2) + "\n" to path.

5. **`fetch_gnews_query(query: str, api_key: str, quota: GNewsQuota) -> tuple[list[Article], GNewsQuota, str | None]`** — Per RESEARCH.md Pattern 3:
   - If quota.calls_used >= quota.daily_limit: log WARNING, return ([], quota, "quota exhausted")
   - Build params: q=query, lang="en", country="in", max=10, apikey=api_key
   - Call httpx.Client(timeout=15.0).get(GNEWS_BASE, params=params)
   - On success: parse JSON, normalize each article to Article (source="GNews", published_at from article's "publishedAt" ISO field, summary="" always, fetched_at=now_iso). Increment calls_used. Return (articles, new_quota, None).
   - On 401: log ERROR "GNews auth failure — check GNEWS_API_KEY". Return ([], quota, "auth failure 401"). Do NOT increment calls_used.
   - On 429: log WARNING, set calls_used=daily_limit. Return ([], exhausted_quota, "rate limited 429").
   - On other HTTPStatusError: log ERROR. Return ([], quota, f"HTTP {status_code}").
   - On network/other exceptions: log ERROR, return ([], quota, error_string). Use `# noqa: BLE001` on broad Exception.

6. **`fetch_all_gnews(queries: list[str], api_key: str, quota: GNewsQuota) -> tuple[list[Article], GNewsQuota, list[dict]]`** — iterate queries, call fetch_gnews_query for each, accumulate articles. Build health_results list: `{"query": q[:50], "status": "OK"/"FAIL"/"SKIP", "count": N, "error": err}`. Stop early if quota exhausted (remaining queries get status "SKIP"). Return (all_articles, final_quota, health_results).
