<task type="auto" tdd="true">
  <name>Task 1: Telegram API sender with retry and deliver orchestrator</name>
  <files>src/pipeline/deliverers/telegram_sender.py, tests/test_telegram_sender.py</files>
  <behavior>
    - send_telegram_message(token, chat_id, text) -> (True, None) on HTTP 200 with {"ok": true}
    - send_telegram_message(token, chat_id, text) -> (False, "rate limited (429)") on HTTP 429
    - send_telegram_message(token, chat_id, text) -> (False, "HTTP 400") on other HTTP errors
    - send_telegram_message(token, chat_id, text) -> (False, "connection error message") on network failure
    - send_telegram_message retries once on 429 with 2-second delay, returns success if retry works
    - send_telegram_message retries once on network error with 2-second delay
    - deliver_articles(articles, config) with valid token sends to all chat_ids, returns delivery count
    - deliver_articles(articles, config) with empty token logs warning and returns 0
    - deliver_articles(articles, config) with empty chat_ids logs warning and returns 0
    - deliver_articles calls select_articles, format_delivery_message, then send_telegram_message for each chunk per chat_id
    - deliver_articles logs delivery summary (articles selected, messages sent, failures)
  </behavior>
  <action>
    Add sender and orchestrator functions to `src/pipeline/deliverers/telegram_sender.py` (appending to the formatting functions from Plan 01).

    1. `send_telegram_message(token: str, chat_id: str, text: str) -> tuple[bool, str | None]`:
       - POST to `https://api.telegram.org/bot{token}/sendMessage`
       - Payload: `{"chat_id": chat_id, "text": text, "parse_mode": "HTML", "link_preview_options": {"is_disabled": True}}`
       - Use `httpx.Client(timeout=15.0)` (consistent with project pattern from gnews_fetcher.py)
       - On 429: single retry after 2-second `time.sleep(2)`, then return failure if still 429
       - On network error (httpx.ConnectError, httpx.TimeoutException): single retry after 2 seconds
       - On other HTTP errors: return (False, "HTTP {status}")
       - On success with {"ok": true}: return (True, None)
       - On success with {"ok": false}: return (False, description)

    2. `deliver_articles(articles: list[Article], config: AppConfig) -> int`:
       - Get token from `os.environ.get("TELEGRAM_BOT_TOKEN", "")` or `config.telegram.bot_token` (env var takes precedence)
       - Get chat_ids from `os.environ.get("TELEGRAM_CHAT_IDS", "")` — if set, split by comma; otherwise use `config.telegram.chat_ids`
       - If token empty: log warning "TELEGRAM_BOT_TOKEN not set -- skipping delivery", return 0
       - If chat_ids empty: log warning "No Telegram chat IDs configured -- skipping delivery", return 0
       - Call `select_articles(articles, config.delivery.max_stories)` to get (high, medium, low)
       - If all three empty: log info "No articles to deliver", return 0
       - Call `get_delivery_period()` to determine "Morning" or "Evening"
       - Call `format_delivery_message(high, medium, low, period)` to get list of message chunks
       - For each chat_id, for each chunk: call `send_telegram_message`, add 0.5s delay between sends
       - Log delivery summary: "Delivered {n} articles in {m} messages to {c} users ({f} failures)"
       - Return total successful sends

    Add `time` import for sleep. Add `os` import for env vars. Import `select_articles` from `.selector`. Import `AppConfig` from `pipeline.schemas.config_schema`.

    Add tests to `tests/test_telegram_sender.py` using respx to mock the Telegram API endpoint. Follow the project's respx pattern from test_gnews_fetcher.py. Use `@respx.mock` decorator. Mock `time.sleep` with `unittest.mock.patch` to avoid real delays in tests.
  </action>
  <verify>
    <automated>uv run pytest tests/test_telegram_sender.py -x -v</automated>
  </verify>
  <done>send_telegram_message handles success, 429, HTTP errors, and network failures with single retry. deliver_articles orchestrates selection, formatting, and sending with proper logging. Graceful skip when token/chat_ids missing. All tests pass.</done>
</task>
