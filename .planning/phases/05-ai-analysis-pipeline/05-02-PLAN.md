    - When Claude call raises any exception, Gemini is attempted with identical prompt
    - When both Claude and Gemini fail, articles get priority="MEDIUM", empty summary and entities
    - Articles are truncated before sending: title max 200 chars, summary max 500 chars

  <action>
    1. Create `src/pipeline/analyzers/classifier.py` with:

       **Constants:**
       - CLAUDE_MODEL = "claude-haiku-4-5"
       - GEMINI_MODEL = "gemini-2.5-flash"

       **System prompt (SYSTEM_PROMPT constant):**
       - Domain-primed for Indian infrastructure and real estate news classification
       - Classification criteria from CONTEXT.md:
         - HIGH: policy/regulatory impact (RERA, PMAY, metro approvals, highway sanctions) AND major project milestones (metro line approvals, airport expansions, smart city awards, large land deals) AND market-moving events (interest rate changes, stamp duty revisions, celebrity transactions, large builder announcements)
         - MEDIUM: progress updates on known projects (construction updates, tender awards, deadline extensions) AND regional/Tier 2-3 developments (new industrial corridors, smart city progress, local RERA actions)
         - LOW: catch-all for commentary, opinions, forecasts, routine operational news, compliance updates
       - Summary style: writer-focused impact summaries for Magic Bricks content writers -- "why this matters for your next article", not generic news summaries. Each summary is exactly 2 lines.
       - Entity extraction: location (city/state/region), project_name (project or scheme name), budget (monetary figure with currency), authority (government body or organization). Empty string if not found.
       - Output format: JSON array matching BatchClassificationResponse schema

       **build_articles_text(articles: list[Article]) -> str:**
       - Format each article as numbered item: "Article {i}: Title: {truncated_title}\nSource: {source}\nSummary: {truncated_summary}\n"
       - Truncate title at 200 chars, summary at 500 chars
       - Log count of truncated articles if any

       **_classify_with_claude(articles_text: str) -> tuple[BatchClassificationResponse | None, dict]:**
       - Create anthropic.Anthropic() client (reads ANTHROPIC_API_KEY from env)
       - Call client.messages.parse() with SYSTEM_PROMPT (using cache_control: ephemeral), articles_text as user message, output_format=BatchClassificationResponse, max_tokens=4096, model=CLAUDE_MODEL
       - Extract usage dict from response.usage (input_tokens, output_tokens, cache_creation_input_tokens, cache_read_input_tokens)
       - Check response.stop_reason == "end_turn" and response.parsed_output is not None (per Research Pitfall 4)
       - Return (parsed_output, usage) or (None, {}) on any failure
       - Wrap in try/except for any Exception -- log and return (None, {})

       **_classify_with_gemini(articles_text: str) -> tuple[BatchClassificationResponse | None, dict]:**
       - Read GOOGLE_API_KEY from env (per Research Pitfall 2, use GOOGLE_API_KEY not GEMINI_API_KEY)
       - If no API key, return (None, {})
       - Create genai.Client(api_key=api_key)
       - Call client.models.generate_content() with model=GEMINI_MODEL, contents=f"{SYSTEM_PROMPT}\n\n{articles_text}", config with response_mime_type="application/json" and response_json_schema=BatchClassificationResponse.model_json_schema()
       - Parse response: BatchClassificationResponse.model_validate_json(response.text)
       - Extract usage from response.usage_metadata (prompt_token_count, candidates_token_count)
       - Return (result, usage) or (None, {}) on any failure
       - Wrap in try/except for any Exception -- log and return (None, {})

       **classify_articles(articles: list[Article], ai_cost: AICost) -> tuple[list[Article], AICost]:**
       - If not articles: return ([], ai_cost)
       - Check budget via check_budget(ai_cost):
         - "exceeded": log warning, return articles with priority="MEDIUM" (map relevance_score: >=80 -> "HIGH", >=60 -> "MEDIUM", else "LOW" as keyword-only fallback), no summary/entities, return (degraded_articles, ai_cost)
         - "warning": log warning, proceed
       - Build articles_text via build_articles_text(articles)
       - Try Claude: result, usage = _classify_with_claude(articles_text)
       - If result is None: try Gemini: result, usage = _classify_with_gemini(articles_text); provider = "gemini"
       - If result is None (both failed): return articles with priority="MEDIUM", empty summary/entities, return (fallback_articles, ai_cost)
       - Map result.articles back to input articles using index field: article.model_copy(update={priority, summary, location, project_name, budget_amount, authority})
       - Handle index mismatches gracefully: unmatched articles get priority="MEDIUM"
       - Record cost: ai_cost = record_cost(ai_cost, usage input_tokens, usage output_tokens, provider)
       - Log classification summary: counts of HIGH/MEDIUM/LOW
       - Return (classified_articles, ai_cost)
